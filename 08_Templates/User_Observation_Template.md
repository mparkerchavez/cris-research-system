## When to Create a User Observation

**Good candidates:**
- Pattern you've seen across multiple projects/organizations
- Mental model or framework that helps you make sense of complex phenomena
- Insight from direct experience that published sources haven't captured
- Rapid developments where your real-time observation precedes formal research

**Not a good fit:**
- Something fully covered by an extraction (just cite the extraction)
- Speculation without experiential basis
- Opinion without observable pattern

## Integration Steps

After adding a new observation:

1. **Update Active Ideas** — If the observation supports or challenges an existing idea, add it to the Supporting Evidence table with `[User observation]` citation

2. **Update Weekly Learnings** — If synthesizing during current week, add to "Personal Observations Integrated" section

3. **Consider Talking Points** — Does this observation change your "Your Angle" on any talking point?

4. **Update _Index.md** — Add to Recent Activity table

5. **Check for new idea emergence** — Does this observation suggest a new Active Idea should be created?
