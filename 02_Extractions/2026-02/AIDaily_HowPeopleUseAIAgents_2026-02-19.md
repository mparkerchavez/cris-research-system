## Data Points

**DP1:** The gap between theoretical agent autonomy (measured by METR) and actual real-world usage is substantial; agents are given far less autonomy than their technical capabilities allow.
- **Anchor:** "real-world AI agents are currently given much less autonomy than they could technically handle"
- **Citation:** 00:49 (David Hendrickson quote)
- **Tags:** #capability-overhang, #deployment-bottleneck, #agentic-workflows

**DP2:** METR's task duration metric measures how long a task would take a human to complete, not how long an agent actually takes; achieving 50-80% success rates is insufficient for real-world employment contexts.
- **Anchor:** "it's how long that task would have taken a human"
- **Citation:** 00:41-02:44
- **Tags:** #measurement-framework-reckoning, #roi-measurement

**DP3:** Autonomy maturity in agent systems follows a trust-building progression: new Claude Code users approve actions 20% of the time, increasing to 40% for experienced users.
- **Anchor:** "a steady accumulation of trust. At the beginning, you approve things each time, and then as you dial in your settings and you start to learn to trust the model, you give it that auto-approval more frequently"
- **Citation:** 06:26-07:38
- **Tags:** #agentic-workflows, #deployment-stages, #human-centered-ai

**DP4:** Experienced Claude Code users interrupt agent work at nearly double the rate of new users (9% vs 5%), suggesting learned pattern recognition for when intervention is valuable.
- **Anchor:** "The higher interrupt rate may also reflect active monitoring by users who have more honed instincts for when their intervention is needed"
- **Citation:** 07:38-08:16
- **Tags:** #human-centered-ai, #intelligent-delegation, #skill-formation

**DP5:** Long-duration agent autonomy (99.9th percentile) in Claude Code increased from 25 minutes (October 2025) to 45 minutes (January 2026), driven by smooth gains across model releases.
- **Anchor:** "average turn duration at that percentile jumped from 25 minutes to 45 minutes"
- **Citation:** 04:20-06:26
- **Tags:** #model-capabilities, #temporal-compression, #agentic-workflows

**DP6:** Recent Claude Code user base doubling (January-February 2026) brought more diverse users with shorter session durations, masking power users' advanced use cases in aggregate data.
- **Anchor:** "the Claude Code user base doubled—which is obviously a phenomenon that we've been tracking closely here. A doubling like that is naturally going to bring with it a more diverse user base"
- **Citation:** 05:00-06:04
- **Tags:** #adoption-barriers, #workforce-readiness, #deployment-stages

**DP7:** As Claude model capabilities improved 2x (August-December 2025), human interventions per session dropped from 5.4 to 3.3, demonstrating that improved model reliability directly enables greater operational autonomy.
- **Anchor:** "as Claude Code's success rate on internal users' most challenging tasks doubled, the average number of human interventions per session decreased from 5.4 to 3.3"
- **Citation:** 08:16-08:38
- **Tags:** #model-capabilities, #productivity-impact, #workflow-optimization

**DP8:** Agents actively participate in autonomy negotiation: Claude Code requests clarification at 2x the rate humans interrupt for complex tasks (16.4% vs 7.1%).
- **Anchor:** "Claude asked for clarification more than double that—16.4% of the time"
- **Citation:** 08:38-09:24
- **Tags:** #intelligent-delegation, #agentic-workflows, #model-capabilities

**DP9:** Top reasons for human interruption are missing context/corrections (32%) and performance issues (17%), while agents most frequently stop to present users with approach alternatives (35%).
- **Anchor:** "the number one reason that humans interrupt Claude is to provide missing context or corrections; that's 32% of the time"
- **Citation:** 09:24-10:07
- **Tags:** #human-centered-ai, #collaboration-paradox, #workflow-optimization

**DP10:** Software engineering accounts for 50% of agent tool calls in production, but 50%+ of usage is already distributed across back office (9.1%), marketing (4.4%), sales CRM (4.3%), and finance/accounting (4.0%).
- **Anchor:** "software engineering represents around half of the tool calls overall. And although the other categories are all below 10%, they kind of read like a map of where agentic automation is likely to come next"
- **Citation:** 10:07-10:50
- **Tags:** #domain-specific-agents, #market-demand, #use-case-development, #workflow-transformation

**DP11:** Claude Code represents product-market fit for agents by functioning as a code-enabled general-purpose tool rather than narrow code editor.
- **Anchor:** "Claude Code is better thought of not as a coding tool per se, but instead as a code-enabled general-purpose agent"
- **Citation:** 02:44-03:07
- **Tags:** #agent-native-development, #product-strategy, #agentic-workflows

**DP12:** METR's idealized testing methodology (no human interaction, no real-world consequences) fails to capture actual deployment contexts where human oversight and user expertise shape what agents can accomplish.
- **Anchor:** "the METR evaluation captures what a model is capable of in an idealized setting with no human interaction and no real-world consequences"
- **Citation:** 02:30-02:44
- **Tags:** #measurement-framework-reckoning, #deployment-bottleneck, #human-centered-ai

**DP13:** Agent autonomy cannot be defined by model capability alone; it emerges from the interaction state between model, human supervisor, and operational context.
- **Anchor:** "when we try to understand agent autonomy, we have to think beyond just the model to the entire context in which a model operates, including the human interactive context"
- **Citation:** 04:20-06:26
- **Tags:** #measurement-framework-reckoning, #agentic-workflows, #workflow-transformation

**DP14:** User demand exists for "competent autonomy" — agents that skip redundant approval prompts while respecting safety boundaries, suggesting current interfaces over-throttle available capability.
- **Anchor:** "Need a Claude Code mode that isn't exactly 'dangerously skip permissions' but can skip pointless 'do you want to proceed' questions, and at the same time doesn't nuke my entire database"
- **Citation:** 10:50-11:03
- **Tags:** #agentic-burden, #workflow-optimization, #risk-governance

**DP15:** The next wave of agent development may diverge between improved human-agent interaction paradigms versus enabling long-duration autonomy (6+ hour independent work cycles).
- **Anchor:** "the next leap in AI isn't just smarter models but long-duration autonomy. While today's tools are optimized for short bursts, tomorrow's tools will be agents you dispatch for 6 plus hours"
- **Citation:** 11:03-11:38
- **Tags:** #product-strategy, #temporal-compression, #agentic-workflows, #deployment-stages

**DP16:** Median Claude Code session duration remains stable at 45 seconds despite user base expansion and capability improvements, indicating task type and user workflow patterns determine practical autonomy horizons.
- **Anchor:** "most Claude Code turns are very short. The median turn lasts around 45 seconds, and that's been fairly consistent over the past several months"
- **Citation:** 04:20-05:00
- **Tags:** #ambient-work, #workflow-transformation, #agentic-workflows
