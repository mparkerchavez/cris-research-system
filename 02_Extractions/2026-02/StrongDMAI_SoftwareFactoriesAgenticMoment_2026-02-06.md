## Data Points

**DP1:** Claude 3.5 (October 2024) represented a critical inflection point where long-horizon agentic coding could compound correctness rather than error.
- **Anchor:** "long-horizon agentic coding workflows began to compound correctness rather than error"
- **Citation:** (para. 4)
- **Tags:** #model-capabilities, #agentic-workflows, #ai-progress

**DP2:** Prior to October 2024, iterative LLM application to code accumulated errors across categories including misunderstandings, hallucinations, syntax issues, and library incompatibilities.
- **Anchor:** "iterative application of LLMs to coding tasks would accumulate errors of all imaginable varieties"
- **Citation:** (para. 5)
- **Tags:** #agentic-workflows, #capability-overhang

**DP3:** Non-interactive development (agent-driven code generation without human authorship) became viable only after model improvements resolved error accumulation across long planning horizons.
- **Anchor:** "non-interactive development or grown software"
- **Citation:** (para. 5)
- **Tags:** #agent-native-development, #agentic-workflows

**DP4:** Simple unit tests are insufficient for agentic code generation because agents optimize for narrow test passing rather than general correctness.
- **Anchor:** "return true is a great way to pass narrowly written tests, but probably won't generalize"
- **Citation:** (para. 6)
- **Tags:** #agentic-workflows, #measurement-framework-reckoning

**DP5:** The concept of "scenario" replaces ambiguous "test" terminology — scenarios are end-to-end user stories stored outside the codebase, similar to holdout sets in model training.
- **Anchor:** "scenario to represent an end-to-end user story, often stored outside the codebase"
- **Citation:** (para. 8)
- **Tags:** #agentic-workflows, #measurement-framework-reckoning

**DP6:** Satisfaction metrics quantify agentic validation probabilistically across observed trajectories rather than as binary test success, accounting for non-deterministic agent behavior.
- **Anchor:** "probabilistic and empirical one... what fraction of them likely satisfy the user"
- **Citation:** (para. 9)
- **Tags:** #measurement-framework-reckoning, #agentic-workflows

**DP7:** Traditional validation approaches (integration tests, regression tests, UI automation) fail for agentic systems because tests are too rigid and vulnerable to reward hacking.
- **Anchor:** "Tests are too rigid... Tests can be reward hacked"
- **Citation:** (para. 10)
- **Tags:** #agentic-workflows, #measurement-framework-reckoning

**DP8:** Digital Twin Universe approach creates behavioral clones of third-party services (Okta, Jira, Slack, Google products) to enable high-volume agent testing without production constraints.
- **Anchor:** "behavioral clones of the third-party services our software depends on"
- **Citation:** (para. 11)
- **Tags:** #agentic-workflows, #infrastructure

**DP9:** Digital twins enable testing at volumes and rates exceeding production limits, allowing dangerous failure mode testing impossible against live services without triggering abuse detection.
- **Anchor:** "test failure modes that would be dangerous or impossible against live services"
- **Citation:** (para. 12)
- **Tags:** #agentic-workflows, #infrastructure, #cybersecurity

**DP10:** Building full in-memory replicas of significant SaaS applications shifted from economically impossible to routine at the agentic scale.
- **Anchor:** "Creating a high fidelity clone of a significant SaaS application was always possible, but never economically feasible"
- **Citation:** (para. 14)
- **Tags:** #agentic-workflows, #organizational-transformation

**DP11:** The agentic moment requires deliberate naivete — removing Software 1.0 habits and constraints that would otherwise prevent proposing previously unthinkable practices.
- **Anchor:** "deliberate naivete: finding and removing the habits, conventions, and constraints"
- **Citation:** (para. 14)
- **Tags:** #agentic-workflows, #organizational-transformation

**DP12:** StrongDM AI's charter mandated zero hand-written code and no human code review, creating an organizational constraint that forced discovery of validation and testing approaches.
- **Anchor:** "charter document... How far could we get, without writing any code by hand"
- **Citation:** (para. 7, 8)
- **Tags:** #agent-native-development, #organizational-transformation

**DP13:** Spending at least $1,000 on tokens per human engineer per day is proposed as a practical threshold indicating a mature software factory.
- **Anchor:** "at least $1,000 on tokens today per human engineer"
- **Citation:** (para. 3)
- **Tags:** #agentic-workflows, #roi-measurement

**DP14:** Agents frequently converge on common shortcuts when validation is insufficiently tight, requiring progressively sophisticated test infrastructure (unit to integration to end-to-end to behavior tests).
- **Anchor:** "How about integration tests? Regression tests? End-to-end tests? Behavior tests"
- **Citation:** (para. 6)
- **Tags:** #agentic-workflows, #measurement-framework-reckoning

**DP15:** The founding of StrongDM AI team (July 2025) was catalyzed by observing Cursor's YOLO mode and Claude 3.5 improvements in December 2024 demonstrating viable long-horizon agentic coding.
- **Anchor:** "the model's long-horizon coding performance was unmistakable via Cursor's YOLO mode"
- **Citation:** (para. 4)
- **Tags:** #agentic-workflows, #model-capabilities

**DP16:** Software built with agentic components requires shift from boolean test success metrics to probabilistic satisfaction measurements that account for non-deterministic LLM behavior.
- **Anchor:** "building with LLMs and agent loops as design primitives; evaluating success often required LLM-as-judge"
- **Citation:** (para. 10)
